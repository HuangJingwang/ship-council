# Publish Quality

Use this reference when preparing Change Crew for GitHub or a marketplace.

## Skill Quality

- Keep `SKILL.md` short and route to references.
- Keep the description focused on trigger conditions, not process summary.
- Do not include private project names or local paths in reusable instructions.
- Prefer scripts for deterministic checks.
- Make templates easy to copy and grep.
- Include realistic examples in the public README, not inside the installed skill unless needed.

## Repository Quality

- First README screen must state what the skill does, who it is for, and one example.
- Include install instructions for Codex and generic `.agents/skills`.
- Mention constraints honestly: multi-agent workflows cost more tokens and need verification commands.
- Do not promise autonomous correctness.
- Use MIT or another clear license.
